Installation

    Installing Angular CLI using the npm package manager
  • npm install -g @angular/cli
    unstalling Angular CLI using the npm package manager
  • npm uninstall -g @angular/cli
    To create, build, and serve a new, basic Angular project on a development server, go to the parent directory of your new workspace use the following commands:
  • ng new my-first-project
  • cd my-first-project
  • ng serve / ng serve --host 0.0.0.0(example ng serve -host 10.0.90.30)
    To generate components,services,modules,pipe
  • ng generate component User / ng g component User
  • ng generate service User / ng g service User
  • ng generate module User / ng g module User
    To build project
  • ng build --prod
    1. Source mapping - Exact each file location can be identified if it is true.(In prod by default is false)
    2. Minification â€“ removing all the white spaces (compressing).
    3. Uglification - changing all the variables and function names while compressing.
    4. Tree shaking - excludes all the un used component while building
    5. AOT - The AOT compiler detects and reports template binding errors during the build step before users can see them. AOT compiles HTML templates and components into JavaScript files long before they are served to the client. With no templates to read and no risky client-side HTML or JavaScript evaluation, there are fewer opportunities for injection attacks.
